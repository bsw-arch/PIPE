# PIPE Task Bot - GitFlow Workflow

## Branch Structuur

```
main (production)
  ↑
develop (integration)
  ↑
feature/* (nieuwe features)
```

## Workflow voor Nieuwe Features

### 1. Start Feature Branch
```bash
git checkout develop
git pull origin develop
git checkout -b feature/wolfi-base-image
```

### 2. Ontwikkel Feature
```bash
# Maak wijzigingen
git add .
git commit -m "feat(base): implement Wolfi base image config"
```

### 3. Push Feature Branch
```bash
git push -u origin feature/wolfi-base-image
```

### 4. Merge naar Develop
```bash
git checkout develop
git pull origin develop
git merge feature/wolfi-base-image
git push origin develop
```

### 5. Merge naar Main (via PR)
```bash
# Maak Pull Request: develop → main
# Na approval:
git checkout main
git pull origin main
git merge develop
git push origin main
```

## Branch Naming Convention

- `feature/wolfi-configs` - Nieuwe feature
- `bugfix/container-size` - Bug fix
- `hotfix/security-patch` - Critical fix
- `docs/wolfi-guide` - Documentation

## Huidige Status

✅ **main** - Production (huidige structuur)
✅ **develop** - Integration (sync met main)
⏳ **feature/** - Volgende features hier

## Volgende Feature

Voor Wolfi/APKo implementatie:
```bash
git checkout develop
git checkout -b feature/wolfi-apko-configs
# ... implementeer configs ...
git push -u origin feature/wolfi-apko-configs
# PR: feature → develop → main
```
